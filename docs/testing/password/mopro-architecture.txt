Current proof flow
1. Android/Kotlin
   - LocationProofComponent.kt obtains zkey paths with getFilePathFromAssets().
   - getFilePathFromAssets() in MainActivity.kt copies an asset stream to context.filesDir and returns a normal filesystem path.
   - Proof operations run in startProofWorker() with a 64 MiB Java thread stack.

2. Rust/FFI
   - Kotlin calls generated UniFFI functions in uniffi.mopro.
   - generateCircomProof() enters Rust generate_circom_proof().
   - verifyCircomProof() enters Rust verify_circom_proof().

3. Input construction
   - Location input:
     - Kotlin calls generateLocationCircuitInput().
     - Rust h3_converter::generate_circuit_input() computes H3 bounds and Poseidon(x, y, salt).
     - normalize_rust_witness_input() wraps scalar strings into arrays for rust_witness compatibility.
   - Single-field regex input:
     - Kotlin directly builds {"msg":[ascii bytes]} in LocationProofComponent.kt.
   - Regex record input:
     - Kotlin parses JSON in RegexRecordProof.kt.
     - Rust h3_converter::generate_regex_record_circuit_input() normalizes fields, packs bytes, computes Poseidon(10), and returns a nested circuitInput object.

4. Witness generation
   - Android target:
     - build.rs copies repo wasm files into OUT_DIR/android-witness-wasm and calls rust_witness::transpile::transpile_wasm().
     - src/lib.rs registers rust_witness generated functions:
       - areajudge_witness
       - regexip_witness
       - regextimestamp_witness
       - porttrans_witness
       - unit_witness
       - protocolregex_witness
       - regexrecord_witness
   - Non-Android target:
     - build.rs calls witnesscalc_adapter::build_and_link("./test-vectors/circom/witnesscalc").
     - src/lib.rs currently registers only areajudge and regex_ip for desktop non-Android.

5. Proof generation
   - generate_circom_proof() extracts the zkey file name from the passed path.
   - It uses circom_get(file_name) from set_circom_circuits! to select the registered witness function.
   - It calls CircomProver::prove(proof_lib, witness_fn, circuit_inputs, zkey_path).
   - The result is converted to UniFFI-friendly CircomProofResult with proof a/b/c and public inputs.

6. Local verify
   - Android calls verifyCircomProof(zkeyPath, proofResult, ProofLib.ARKWORKS).
   - Rust verify_circom_proof() calls CircomProver::verify(proof_lib, proof, zkey_path).
   - Therefore Android local verification currently also uses the zkey path, not verification_key.json.
